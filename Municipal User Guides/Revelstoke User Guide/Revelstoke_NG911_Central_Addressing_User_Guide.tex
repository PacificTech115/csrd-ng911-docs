\section{Purpose and scope}
This guide explains the standard Revelstoke workflow for editing the NG911 central addressing data:
\begin{enumerate}
\item Log in to the CSRD portal.
\item Open the Revelstoke NG911 group.
\item Open the \texttt{NG911\_Address\_Revelstoke\_Edit} feature layer.
\item Choose \textbf{Open in ArcGIS Pro}.
\item Edit attributes/geometry and save edits.
\item Verify changes in the Feature Layers Data tab on the CSRD Portal.
\end{enumerate}

\subsection{Role guidance}
\begin{itemize}
\item Use the \textbf{Viewer} account for read-only access, review, and validation.
\item Use the \textbf{Editor} account for production edits to the NG911 address layer.
\item Only perform address edits in the designated editing layer item.
\end{itemize}

\section{Quality checklist before save}
\begin{itemize}
\item Correct layer: \texttt{NG911\_Address\_Revelstoke\_Edit}
\item Correct account: Editor account for edits
\item Required fields populated (agency, address number, street name, locality)
\item No obvious typos in civic address or locality
\item Point location is accurate
\end{itemize}

\section{Troubleshooting}
\subsection{Cannot sign in}
\begin{itemize}
\item Confirm username/password are entered exactly (case-sensitive).
\item Ensure you are using the correct role account (Viewer vs Editor).
\item Retry sign-in from the CSRD portal home page.
\end{itemize}

\subsection{Open in ArcGIS Pro does not launch}
\begin{itemize}
\item Ensure ArcGIS Pro is installed on the workstation.
\item Ensure the machine is associated with the correct portal/account access.
\item Try launching ArcGIS Pro first, then re-click \textbf{Open in ArcGIS Pro}.
\end{itemize}

\subsection{Edits fail to save}
\begin{itemize}
\item Verify you are signed in with the Editor account.
\item Check required fields and geometry validity.
\item Confirm the layer is editable and not opened read-only.
\end{itemize}
